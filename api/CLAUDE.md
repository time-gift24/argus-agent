# API CLAUDE.md

本目录是前后端共享的接口契约源，当前以 `openapi.yaml` 为主。

## 目录职责

- 定义稳定的 HTTP API 契约（路径、方法、参数、响应结构、错误码）
- 作为 backend 与 frontend 的共同“事实来源”
- 降低前后端并行开发时的误解和返工

## 当前文件

- `openapi.yaml`: OpenAPI 3.0.3 契约文件

## 变更规则（必须遵守）

1. 后端新增/修改/删除接口时，同步更新 `openapi.yaml`。
2. 前端若按新字段开发，必须确认契约已更新，不允许“代码先行、契约滞后”。
3. 破坏性变更（删字段、改语义、改错误码）要在描述中明确标注影响范围。

## 编写要求

- 路径命名清晰一致，避免同义路径并存。
- 字段命名保持稳定，新增优于重命名。
- 对鉴权、必填项、默认值、错误响应写清楚，不靠实现猜测。

## PR 自检清单

- `openapi.yaml` 是否与 backend 实际路由一致（含 tools / mcp-configs / providers 等）？
- frontend 使用的请求路径与字段是否能在契约中找到？
- 示例响应是否可被前端直接消费（字段类型与 nullable 语义一致）？
